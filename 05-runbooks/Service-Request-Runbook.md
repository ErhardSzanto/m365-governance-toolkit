# Service Request Runbook (M365)

## Purpose
Handle common user requests consistently and in line with governance standards.

## Common request types
- New Team / SharePoint site request
- Add/remove members or owners
- Guest access request
- Sharing link issues
- Restore files / recover content (where applicable)
- Request for exceptions to standards (time-bound)

## Intake requirements (minimum information)
- Requester name + department
- Business purpose
- Proposed owner(s)
- Sensitivity level (Public/Internal/Confidential/Highly Confidential)
- External sharing need (yes/no; guest domains; duration)
- Target date / urgency

## Fulfilment steps (generic)
1. Validate request completeness
2. Check against standards (naming, lifecycle, sharing)
3. If exception required:
   - complete exception template
   - obtain approval
   - set expiry date and review reminder
4. Provision or execute request
5. Confirm outcome with requester
6. Update ticket/backlog with completion notes

## Time targets (example)
- Standard requests: 2–5 business days
- Guest access with approval: 3–7 business days
- Exceptions: case-by-case, but always documented

## Close-out checklist
- Ownership assigned
- Labels guidance provided
- Guest expiry set (if applicable)
- Documentation updated if new pattern discovered
